# 필름 데이트 스탬프 (쿼츠 데이트백) — 물리 과정 재현

## 핵심 사상

이 기능은 날짜 텍스트를 사진 위에 "얹는" 것이 아니라, **실제 필름 카메라의 쿼츠 데이트백이
날짜를 필름에 각인하는 물리적 과정 그 자체를 재현**한다. 데이트백은 필름 뒤편의 작은 LED
어레이가 셔터가 열릴 때 필름 에멀전의 한쪽 구석을 **빛으로 노광**하는 장치다. 즉 날짜는
"사진 위의 스티커"가 아니라 **사진과 똑같은 에멀전에 빛으로 새겨진, 사진의 일부**다.

그래서 구현의 모든 선택은 "이 각인이 실제 필름에서 어떻게 보였을까"에서 출발한다 —
합성 방식, 그레인, 번짐, 색, 위치, 해상도까지.

---

## 실제 물리 과정

1. **광원**: 데이트백의 소형 7-세그먼트(또는 14-세그) LED가 호박색(주황) 빛을 낸다.
2. **노광(가산)**: 셔터가 열리면 그 빛이 필름 에멀전의 한 구석을 이미 맺힌 상(像) **위에 더해**
   노광한다. 빛을 더하는 것이므로,
   - 이미 밝은(과노출된) 하늘/하이라이트 위에서는 더 밝아질 여지가 없어 **흰빛으로 씻겨
     거의 안 보인다**.
   - 어두운 영역 위에서는 주황 각인이 **선명하게** 드러난다.
3. **같은 에멀전**: 날짜도 사진과 동일한 은염 결정에 맺히므로 **같은 필름 그레인**을 탄다
   (디지털처럼 매끈할 수 없다).
4. **할레이션**: 강한 점광원이 에멀전·베이스에서 산란·역반사되어 글자 둘레에 **적색 계열의
   부드러운 번짐(헤일로)** 을 만든다.
5. **프레임 기준 위치**: LED는 **센서(가로) 프레임의 우하단**에 고정 각인한다. 세로로 촬영하면
   (카메라를 회전) 업라이트로 볼 때 각인이 90° 돌아간 채 대응 코너로 이동한다.
6. **현상 후 최종 상(像)**: 각인은 필름에 노광된 뒤 현상되어 나온다 — 즉 크롭/방향이 반영된
   **최종 프레임**에 존재하며, 프레임 크기에 비례한 위치·크기를 갖는다.

## 물리 현상 → 구현 매핑

| 물리 현상 | 구현 | 위치 |
|-----------|------|------|
| **LED가 빛을 더해 노광**(가산) | **screen 합성** `out = 1−(1−배경)(1−색×강도)`. 순수 screen 은 하이라이트에서 과하게 사라지므로 source-over 를 소량 섞어(`SCREEN_MIX`) 완화 → 밝은 곳 옅어짐/어두운 곳 선명함은 유지하되 과다 소멸만 억제 | `date_stamp.stamp_export`(export) · `shaders/stamp.frag`(preview) |
| **같은 에멀전의 그레인** | 스탬프 그레인 = **사진 필름 그레인에 연동**(`grainAmt × STAMP_GRAIN_K`). 사진 그레인을 올리면 각인도 같이 거칠어지고, 그레인 0이면 각인도 매끈 | `render_sprite`(grain) ← `grainAmt` |
| **할레이션(적색 산란 번짐)** | 다중 반경 가우시안 글로우(코어·근접·원거리) + 가로 팽창 밴드로 균일한 사이드 글로우. 색은 **핫코어(밝은 노랑-흰) → 앰버 → 적주황 헤일로** 그라데이션 | `render_sprite`(`C_CORE/C_MID/C_HALO`, gaussian/`grey_dilation`) |
| **7-세그/14-세그 LED 숫자** | DSEG 세그먼트 디스플레이 폰트(정품 재현) — 7-seg/14-seg Classic 의 Regular/Bold × 정체/이탤릭, 그리고 도트매트릭스(Doto 원형 도트) | `STYLES`(fonts/*.ttf, SIL OFL) |
| **프레임 기준 각인 위치** | EXIF Orientation → 센서 우하단이 업라이트 프레임에서 오는 코너로 스프라이트를 회전·배치(세로 사진은 90° 회전 코너) | `rot_from_orientation` / `corner_for_rot` / `_rotate_sprite` |
| **현상 후 최종 프레임에 존재** | 크롭·회전이 끝난 **최종 프레임**에 합성. 위치·크기는 **짧은 변 대비 비율**(크롭해도 코너에 일정 비율로 붙음). 셰이더 프록시에 굽지 않아 **크리스프** 유지 | 프리뷰=cropClip 오버레이 / export=최종 배열 |

## 프리뷰 = Export 정합

각인이 "빛을 더하는" 것이므로 합성은 **배경 픽셀을 읽어야** 한다(어두운/밝은 배경에서 다르게
보이므로). 프리뷰와 export 가 **동일한 screen 수식**을 쓰도록 다음과 같이 맞춘다.

- **Export(numpy)**: `stamp_export` 가 최종 배열 코너에서 배경을 읽어 screen(+source-over 혼합) 합성.
- **Preview(GPU)**: `stampOverlay` 를 **ShaderEffect**(`shaders/stamp.frag`)로 두고, 배경
  (`canvasHolder` = 파이프라인 결과)을 샘플러로 읽어 export 와 같은 수식으로 합성. 스탬프
  스프라이트는 별도 고해상도 텍스처로 유지해 **크리스프**하다.
- 두 경로 모두 `strength(STAMP_STRENGTH)`·`SCREEN_MIX` 등 동일 상수를 사용한다(두 곳 동기).

> ⚠️ **설계 트레이드오프**: 예전에는 셰이더가 스탬프를 직접 합성했으나(정확한 배경-읽기 screen),
> 프록시 해상도라 흐릿하고 크롭 전 좌표라 위치가 어긋났다. 그래서 스탬프를 **최종 프레임
> 오버레이**로 옮겼다. 대신 오버레이가 배경을 못 읽어 한때 source-over(스티커) 로 근사했는데,
> 지금은 오버레이 자체를 **배경을 읽는 ShaderEffect** 로 만들어 "크리스프 + 최종 프레임 위치 +
> 진짜 배경-읽기 screen" 을 모두 달성했다.
>
> ⚠️ **프리뷰 스프라이트 해상도**: 프리뷰 스프라이트는 프록시 기준(`ref_short≈1000`)이라 화면
> 확대 시 다소 소프트하게 보일 수 있다(프록시 근사). Export 는 풀해상도라 각인이 선명하다.

## 사용자 조절 / 고정 파라미터

**사용자 조절(이미지별 저장, 더블클릭 리셋):**
- **Style** — 폰트 방식 9종(7-seg/14-seg × Regular/Bold × 정체/이탤릭, + Dot-matrix).
- **Stamp size** — 숫자 높이/짧은 변 비율(1.2–5.0%, 기본 3.2%).
- **Margin** — 코너 안쪽 여백/짧은 변 비율(0–10%, 기본 5%).
- **날짜 텍스트** — 기본은 EXIF 촬영일(`'YY MM DD`), 직접 편집 가능.
- **On/Off** — `C` 키 / 체크박스.

**물리 재현을 위해 고정(사용자 노출 안 함):**
- **밝기** `STAMP_BRIGHTNESS = 0.85`, 합성 강도 `STAMP_STRENGTH = 0.92`.
- **합성 혼합** `SCREEN_MIX = 0.7`(screen 70% + source-over 30%).
- **그레인** `STAMP_GRAIN_K = 0.24`(사진 `grainAmt` 에 연동 — 독립 노출 안 함).
- **코어 선명도** `CORE_BLUR_FRAC = 0.010`, **할레이션** 글로우 계수/색.

> 조절값을 늘리기보다, "실제 필름이라면 이 값은 사진(에멀전)과 함께 결정된다"는 원칙에 따라
> 그레인은 사진에 연동하고 광학적 성격(밝기·번짐·색)은 고정했다.

## 요약

날짜 스탬프는 "텍스트 오버레이"가 아니라 **필름 에멀전에 LED 빛이 각인되는 물리 과정의 재현**이다:
빛을 더하는 **가산(screen)** 합성으로 어두운 곳엔 선명하고 밝은 곳엔 씻겨 사라지며, **사진과
같은 필름 그레인**을 타고, 강한 빛의 **할레이션**으로 적색 번짐을 남기고, **센서 프레임 기준**
코너에 방향을 반영해 놓이며, **현상된 최종 프레임**에 크리스프하게 존재한다 — 그리고 프리뷰와
export 가 같은 수식으로 이 과정을 재현한다.
