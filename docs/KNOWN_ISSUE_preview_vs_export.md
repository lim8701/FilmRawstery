# 알려진 문제: 프리뷰(GPU) ≠ Export 색 차이 (미해결)

## 증상
- 동일 RAF·동일 보정값에서 **화면 프리뷰**와 **저장된 export 이미지**의 색이 다르게 보인다.
- 대표 사례: `DSCF8035.RAF` — 노란 오토바이/갈색 가방 부분이 **프리뷰에서 더 붉고 채도가 높게**,
  export에서는 상대적으로 차분하게 나온다.
- **휘도(밝기)는 거의 동일**하고, 차이는 **채도/붉은기**에 집중된다.
  - 측정값(같은 크기 ~80px 크롭 기준): 프리뷰 채도 0.346 / R·G비 1.296,
    export(크롬) 0.282 / 1.193, export(윈도우 사진) 0.255 / 1.174.

## 배제한 원인 (테스트로 확인 완료)
| 가설 | 결과 |
|------|------|
| 파이프라인 수식/계수 불일치 | ❌ 아님. 셰이더(`adjust.frag`)와 `pipeline.render_full`(numpy)이 동일 단계·계수. scene-linear 픽셀 단위 상관 0.9997, 평균 ~0.6% 일치 |
| 렌즈보정 단계·auto-exposure 적용 순서 | ✅ **수정함**(프록시가 1.37× 밝던 버그 해결, 커밋 a18008b). 이후에도 채도 차이는 잔존 |
| tone_zones 마스크 기준 divergence | ✅ **수정함**(export도 neutral claBlur 기준 사용) |
| 텍스처 비트깊이 (8bit 프록시) | ❌ 무관. 프록시를 16bit(RGBA64)로 바꿔 재시작 테스트 → 차이 동일 |
| 모니터 색 관리 / 광색역 | ❌ 무관. Windows 디스플레이 프로파일이 **sRGB IEC61966-2.1**(시스템 기본)로 확인 |
| 표시 해상도(작게 보임) | ❌ 무관. 같은 크기로 캡쳐해 비교해도 차이 존재 |
| 처리 해상도(프록시 2560 vs 풀 6246) | ❌ 무관. export를 2560으로 뽑아 프리뷰와 비교해도 차이 존재 |

## 남은(미해결) 원인 추정
- CPU export(numpy)와 GPU export(grabToImage)는 **서로 일치**하며, **둘 다 의도한 알고리즘 결과**다.
- 반면 **화면 프리뷰**(GPU `pipe` → `ShaderEffectSource` → 화면 합성 경로)만 더 채도가 높다.
- 즉 **같은 데이터·같은 셰이더**인데 **GPU 실시간 표시 경로**가 numpy/grab과 미묘하게 다른 결과를 낸다.
- 이는 GPU 드라이버의 수치/텍스처 처리 또는 표시(ShaderEffectSource·레이어·스케일) 경로 고유의
  차이로 보이며, **헤드리스에서 GPU 셰이더가 검정으로 렌더되고 numpy로 재현이 안 되어** CPU 분석
  만으로는 더 좁힐 수 없음을 확인했다.

## 확실한 사실
1. **export 파일은 콜로리메트릭으로 올바르다** — CPU·GPU export가 일치하고, 의도한 파이프라인 결과다.
2. 차이는 **화면 프리뷰가 약간 더 진하게 보이는** 수준이며, **휘도는 동일**하다.

## 권장 / 향후
- **실용**: 최종 색 판단은 **export 기준**으로. 프리뷰는 편집용 근사로 사용.
- **근본 규명**: 실기에서 **RenderDoc** 등으로 프리뷰 프레임을 캡쳐해 셰이더 입력/출력 실제 값을
  떠야 함(이 환경에서는 불가).
- **임시 보정(stopgap)은 보류**: 원인 불명 상태에서 export에 작위적 채도 보정을 넣는 것은
  CLAUDE.md 원칙(작위적 근사 지양)에 어긋나 권하지 않음.

## 관련 코드
- `raw_loader.py` — `load_proxy`(8bit RGB888 프록시), `load_full`(16bit RGBA64 풀해상도, GPU export용)
- `pipeline.py` — `render_full`(CPU export)
- `shaders/adjust.frag` — 프리뷰/Export 공통 셰이더
- `Main.qml` — `pipe`(프리뷰), `gpuExportLoader`/`pipeFull`(GPU export), 렌더 모드 드롭다운(CPU/GPU)
